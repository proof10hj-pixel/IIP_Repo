# Codex 기본 사용 가이드

## 개요
- Codex는 프로그래밍 작업을 돕기 위한 코드 생성 모델입니다. 자연어로 설명하면 코드 스니펫, 함수, 테스트 등을 생성하거나 보완할 수 있습니다.

## 기본 사용 원칙
- 명확한 문제 설명: 원하는 동작(입력, 출력, 예외 처리)을 구체적으로 적습니다.
- 최소한의 예시 포함: 원하는 함수 시그니처나 간단한 예시 입력/출력을 포함하세요.
- 점진적 요청: 큰 기능은 작은 단위로 나눠 테스트하면서 요청합니다.

## 프롬프트 모범 사례
- 역할 설정: "너는 파이썬 개발자야. 함수 구현을 도와줘."처럼 역할을 지정합니다.
- 입력/출력 포맷: JSON, CSV 등 형식을 명시하면 안정적인 출력이 나옵니다.
- 제약 조건 명시: 성능, 라이브러리 사용 여부, 호환성(예: Python 3.10) 등을 규정합니다.

## 코드 검증 및 안전성
- 항상 생성된 코드를 실행 전에 정적 검토하고, 유닛 테스트를 작성해 검증합니다.
- 보안 민감 로직(암호화 키 처리, 인젝션 등)은 자동 생성 코드에 의존하지 말고 전문가 검토를 거치세요.

## 한계와 주의점
- 생성된 코드는 완벽하지 않음: 논리적 오류나 경계 케이스를 놓칠 수 있습니다.
- 최신 라이브러리/환경 정보가 반영되지 않을 수 있으니, 의존성 버전과 호환성을 확인하세요.

## 예시 프롬프트
1. 간단 구현: "파이썬으로 두 숫자를 받아 합을 반환하는 `add(a, b)` 함수를 작성해줘. 예외 처리는 필요 없어." 
2. 제약 포함: "외부 라이브러리 없이 재귀를 사용하지 않고 리스트를 정렬하는 함수를 작성해줘. 시간복잡도 O(n log n) 권장." 

## 권장 워크플로우
1. 문제/요구사항 작성
2. 짧은 프롬프트로 초기 코드 생성
3. 로컬에서 실행·테스트
4. 실패 원인에 맞춰 프롬프트 수정 및 재생성
